# ربات تلگرامی پیش‌بینی سیکل قاعدگی

یه Cloudflare Worker که بر اساس یه طول سیکل ثابت، تاریخ شروع دوره‌ی بعدی رو پیش‌بینی می‌کنه و قبل از رسیدنش (۷، ۳، ۲، ۱ روز مونده و روز خودش) تو تلگرام یادآوری خودکار می‌فرسته. دو زبانه‌ست — انگلیسی و فارسی، که از داخل خود تلگرام انتخاب می‌شه.

## چجوری کار می‌کنه

- یه بار تاریخ آخرین شروع دوره رو به ربات می‌گی (`/setstart`).
- ربات فرض می‌کنه سیکل یه طول ثابت داره (پیش‌فرض ۲۸ روز) و خودش همیشه از روی همون تاریخ جلو می‌ره — لازم نیست هر ماه دستی آپدیتش کنی.
- یه کار زمان‌بندی‌شده (Cron Trigger) هر روز یه بار اجرا می‌شه و چک می‌کنه چند روز تا تاریخ پیش‌بینی‌شده مونده، و رو ۷، ۳، ۲، ۱ روز مونده و روز خودش یه پیام تو تلگرام می‌فرسته.
- داده‌ها (تاریخ شروع، طول سیکل، طول دوره، زبان) تو یه دیتابیس Cloudflare D1 ذخیره می‌شه.

## راه‌اندازی — فقط با داشبورد Cloudflare (بدون نیاز به ترمینال)

### ۱. ساخت ربات تلگرام
- تلگرام رو باز کن، دنبال **BotFather@** بگرد، بزن `/newbot` و مراحل رو دنبال کن.
- BotFather یه **توکن** بهت می‌ده (چیزی شبیه `123456:ABC-...`) — ذخیره‌ش کن، بعداً لازمش داری.
- یه پیام به بات جدیدت بفرست (تا تلگرام یه چت باهات داشته باشه).
- برای پیدا کردن **chat ID** خودت، این آدرس رو تو مرورگر باز کن (جای `<TOKEN>` توکنتو بذار):
  `https://api.telegram.org/bot<TOKEN>/getUpdates`
  دنبال `"chat":{"id": ...}` تو جواب بگرد — همون عدد `CHAT_ID` توئه.

### ۲. ساخت Worker
- برو تو [داشبورد Cloudflare](https://dash.cloudflare.com) → **Workers & Pages** → **Create** → **Create Worker**.
- یه اسم بذار (مثلا `period-tracker-bot`) → **Deploy** بزن (این یه کد پیش‌فرض دیپلوی می‌کنه، مشکلی نیست، قدم بعد عوضش می‌کنیم).

### ۳. کد رو جایگزین کن
- برو تو Worker جدیدت → دنبال دکمه‌ای مثل **Edit code** بگرد (بعضی وقتا تو تب Overview هست).
- هر چی تو ادیتور هست پاک کن، کل محتوای فایل `Worker.js` از این ریپازیتوری رو کپی-پیست کن.
- بزن **Save and Deploy**.

### ۴. ساخت دیتابیس D1
- تو منوی سمت چپ داشبورد Cloudflare، دنبال **Storage & Databases** → **D1 SQL Database** → **Create** بگرد.
- یه اسم بذار (مثلا `period-db`) → **Create**.
- دیتابیس جدید رو باز کن → برو تو تب **Console** → این رو پیست کن و اجرا کن:
  ```sql
  CREATE TABLE IF NOT EXISTS config (key TEXT PRIMARY KEY, value TEXT NOT NULL);
  ```
  (همین کوئری تو فایل `Schema.sql` این ریپازیتوری هم هست)

### ۵. وصل کردن دیتابیس به Worker
- برگرد تو Worker‌ت → تب **Bindings** → **Add** → گزینه **D1 database** رو انتخاب کن.
- دیتابیسی که ساختی رو انتخاب کن.
- **Variable name** رو دقیقاً بذار `DB` (کد همین اسم رو انتظار داره).
- ذخیره کن.

### ۶. اضافه کردن Secrets
- تو Worker‌ت → **Settings** → **Variables and secrets** → **Add**.
- دو تا متغیر از نوع **Secret** اضافه کن:
  - `BOT_TOKEN` — توکنی که از BotFather گرفتی
  - `CHAT_ID` — chat ID‌ای که پیدا کردی

### ۷. اضافه کردن Cron Trigger روزانه
- تو Worker‌ت → **Settings** → **Trigger events** (Cron Triggers) → **Add**.
- برو تو تب **Cron expression** و این رو بنویس:
  ```
  0 9 * * *
  ```
  (یعنی هر روز ساعت ۹ صبح به وقت UTC — اگه می‌خوای ساعت دیگه‌ای باشه، عدد رو عوض کن)
- بزن **Add**.

### ۸. وصل کردن تلگرام به Worker (Webhook)
- این آدرس رو تو مرورگر باز کن (جای `<TOKEN>` توکن بات و جای `<your-worker>` ساب‌دامین واقعی Worker‌ت که تو صفحه‌ی Overview نشون داده می‌شه رو بذار):
  ```
  https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<your-worker>.workers.dev
  ```
- باید `{"ok":true,"description":"Webhook was set"}` ببینی.

### ۹. تنظیم ربات از تو تلگرام
یه چت با ربات باز کن و این‌ها رو به ترتیب بفرست:
```
/start          → انتخاب زبان (انگلیسی یا فارسی)
/setstart YYYY-MM-DD   → آخرین تاریخ شروع دوره که می‌دونی
/setcycle 28    → طول سیکل به روز (اختیاری، پیش‌فرض ۲۸)
/setlength 5    → طول دوره به روز (اختیاری، پیش‌فرض ۵)
/next           → چک کن پیش‌بینی درست به‌نظر میاد
```

همین. از الان به بعد ربات خودش هر روز چک می‌کنه و وقتی دوره‌ی بعدی نزدیک شد بهت پیام می‌ده.

## راه‌اندازی — با Wrangler CLI (روش جایگزین، اگه ترمینال داری)

```bash
wrangler d1 create period-db
# database_id ای که چاپ می‌کنه رو تو Wrangler.toml بذار

wrangler d1 execute period-db --file=./Schema.sql --remote

wrangler secret put BOT_TOKEN
wrangler secret put CHAT_ID

wrangler deploy

curl "https://api.telegram.org/bot<TOKEN>/setWebhook?url=https://<your-worker>.workers.dev"
```

## دستورات ربات

| دستور | توضیح |
|---|---|
| `/start` | انتخاب زبان |
| `/setlang` | تغییر زبان در آینده |
| `/setstart YYYY-MM-DD` | ثبت آخرین تاریخ شروع دوره (یه بار لازمه) |
| `/setcycle N` | تنظیم طول سیکل به روز (پیش‌فرض ۲۸) |
| `/setlength N` | تنظیم طول دوره به روز (پیش‌فرض ۵) |
| `/next` | نمایش تاریخ پیش‌بینی‌شده‌ی بعدی، یا چند روز مونده اگه الان تو دوره‌ای |

ورودی با ارقام فارسی (مثلا `۵`) هم برای دستورات عددی قبول می‌شه.

## فایل‌ها

- `Worker.js` — کد Worker
- `Wrangler.toml` — تنظیمات Worker (`database_id` خودتو بذار توش)
- `Schema.sql` — ساختار جدول D1
